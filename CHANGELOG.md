## 버전 관리 가이드

### Semantic Versioning 규칙
- **MAJOR (X.0.0)**: 호환되지 않는 API 변경
- **MINOR (0.X.0)**: 하위 호환되는 새로운 기능 추가
- **PATCH (0.0.X)**: 하위 호환되는 버그 수정

### 배포 명령어
```bash
# 패치 버전 업데이트 (버그 수정)
npm run version:patch

# 마이너 버전 업데이트 (새 기능)
npm run version:minor

# 메이저 버전 업데이트 (큰 변경사항)
npm run version:major

# 버전 업데이트 후 배포
npm run deploy:version
```
